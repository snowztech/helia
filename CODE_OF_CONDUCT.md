# Code of conduct

Helia is built in the open. We want this project to be a friendly,
useful place to work, ship, and disagree.

## The short version

- **Be kind.** Assume good intent.
- **Be specific.** Vague critiques are noise. Specific critiques move the
  project forward.
- **Disagree about ideas, not people.** "This design has X problem"
  beats "you are wrong."
- **Stay on topic.** Issues are about Helia. Not politics, not
  promotion, not unrelated tooling debates.

## What we will not tolerate

Harassment, personal attacks, sustained disruption of discussion,
publishing others' private information, or any conduct that would be
inappropriate in a professional setting.

## Enforcement

If you see something that violates this code, email
**gethelia@protonmail.com**. We will read every report and respond
proportionally — a warning for a one-off, removal for repeated or
severe behavior.

Maintainers act on their own judgment. We will not litigate every
report publicly. We will keep the project a place where serious people
get useful work done.

## Scope

This applies to all Helia spaces: the GitHub repo, issues, PRs,
discussions, and any community channel under the Helia name.
